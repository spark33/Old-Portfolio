TEF Scheduler

MEAN Stack

Dec 2016 - Mar 2017

I am a personnel manager for CMU's Computing Services. I created the TEF Scheduler to centralize all of TEF's software with enhanced user experience and functionality and replace the old scheduling and reporting software.

This application will be completed this Spring. We expect deployment to occur this Fall after several months of user testing and optimizing for security, dependability and scalability. Check out a demo here! You can log in with ID: seanpark and PW: password.

Introduction

Currently, my department uses WhenToWork to schedule employee shifts, Google Sheets to monitor hour logs and a custom LAMP app for employees to submit reports. Every shift change is made by an administrator who is informed by employees by e-mail, leading to flooded inboxes and many missed shift changes. No employee management system caters to student employees' short shifts and unpredictability in schedules.

Design Research Process

To ensure that the app would improve on all of WhenToWork's pain points, I sent out a survey that collected a mix of general and detailed responses in both quantitative and qualitative forms. I also identified the user stories of an ideal replacement software, which you can find at the bottom of this page. I also conducted competitor analysis to compare their functionalities with the identified user stories.

Shift Transfer Automation

Most employees rated "Making temporary / permanent changes to schedule" as the most difficult interaction. Automating this process became a primary objective of the TEF Scheduler.

UI / UX Improvements

Even employees who had given WhenToWork's convenience and functionalities mid-range ratings gave scathing qualitative responses. Most employees found WhenToWork extremely unattractive and difficult to use.

Competitor Analysis

I analyzed and piloted multiple demos of 4-5 existing scheduling software including Homebase, Shiftboard and Humanity. I noted the convenience of relevant features from both employee and admin perspectives.

After the research was complete, I began to wireframe and prototype the front-end in HTML/CSS/AngularJS. Below, I outline the thinking behind the TEF Scheduler's structure and design.

User Interaction Design

TEF Scheduler was designed to maximize automation and flexibility in scheduling. It makes shifting easier for students while giving administrator a bird's eye view of all shifts and reports. In the future, it will also support data analytics and trend analysis.

The previous model keeps reports completely separate from the scheduling module. The TEF Scheduler merges the reporting app with the scheduling app with shift-specific reports. This allows scalable development of data analysis tools on the same app.

User Profiles

Unlike WhenToWork's separate availability module, the Scheduler does not require the user to open external windows to input availability. All necessary user information can be viewed and edited on one page.

Scheduling Module

On WhenToWork, all functions were done on separate web pages. Shifts had to be manually subbed by deletion and addition by a manager. The new module allows the user to view the schedule by location, pick up shifts, assign subs and submit reports using one simple jQuery calendar. This eliminated extraneous e-mail communication and ~20 clicks for every shift change.

View Reports

By displaying all reports on one view, the manager can simply scroll through reports instead of clicking on individual reports to read them. Reports also send e-mail notifications to relevant staff.

Implementation

To prioritize flexibility in data storage and access, I decided to use a NoSQL database on MongoDB with AngularJS. Given the department's small size, denormalization was not a huge issue. The availability module and scheduler module use the FullCalendar framework. The application is styled using Materialize. The MongoDB database was hosted on MLab and the app was hosted on Heroku. Because this is my first project on the MEAN stack that was built within one month, I am not particularly proud of my code style. I am proud that it is fully functional and will be implemented for CMU Computing Services starting Fall 2017!

Seoul 53 Hotel

AngularJS

Jan 2017

I created the website for Seoul 53 Hotel using AngularJS to handle session data and instant translation.

The development process included designing and prototyping 3 websites. While the client is sorting out domain issues, check out a prototype on Heroku.

Client Interaction



Interaction Method

After assessing my prototype, my clients pointed out details by referring to specific sections using a key. A code would refer to a section on the website, which the client would explain further on an Excel spreadsheet.

Request Examples

Client requests ranged from simple style changes to content additions and restructuring. These changes were made frequently to ensure that the clients were satisfied with the final product.

Design Points

The website was designed to highlight the hotel's cheap prices and proximity to Seoul's tourist attractions. The clean user interface is easy to navigate and pleasant to the eyes. Animation and parallax banners contribute to the website's fluid user experience, and all important information is highlighted to entice users to make reservations. All content is fully mobile responsive. Because it is an image-heavy one-page website, it loads quite slowly; I am still researching how to improve loading speed.

Instant Translation

All site content is automatically translated at the push of a button, without changing pages. All information is handled by JSONs in a single controller so that adding languages and changing page content is easily done without any HTML/CSS knowledge.

Action Cue Placement

Action cues like "Scroll", "Back to Top" and "Reserve" buttons constantly invite users to explore more about the hotel.

Content Writing / Translating

As a trilingual and the only fluent English speaker, I also wrote and translated the website's content. Knowing both languages gave me the client's perspective and the flexibility to adapt content.

HCI Research

Qualitative Data Analysis

Fall 2016

I conducted and presented 3 user interaction research projects for course 05-499C: Designing Large-Scale Peer Learning Systems. I created prototypes for offline interaction systems and analyzed quantitative and qualitative data from interactions to improve future software implementations. Below, I summarize my work from one project from Fall 2016.

If any explanation is insufficient, refer to my attachments at the bottom of this page.

Observing Interactions

This project trained me to make actionable qualitative observations from real-time, in-person interactions. I observed the interactions between my consulting club members to note correlations between the number of side conversations and their personal characteristics (major, grade level, position in club, etc). Below, I briefly summarize my findings using my project slides.

Observations

I noted productive primary discussions, productive side conversations and distracted side conversations using different colored lines. I also noted generalizations in text. Surprisingly, there were close to no correlations between any of the variables, other than their proximity in seating arrangement.

Interview

To find answers, I interviewed several members and board members. Interviewees attributed the lack of correlation to the agenda on Google Documents. Because members were only paying attention while points relevant to them were being discussed, there were many side conversations occurring at random times. Many members stated that side conversations were not hurting the group's productivity.

Conclusion

I concluded that the discussion was layered-- one layer of discussions centralized by the Google Documents, and one layer of side conversations between members sitting next to one another. I connected this finding to previous research on technology enhanced interactions by Jim Hollan and Scott Stornetta.

This Website

HTML/CSS/SASS/JS

Jan 2017 - Mar 2017

This website was created from scratch using HTML5, CSS/SASS and simple JS.

I used Slick for the project slider, Scrollify to paginate and got some animation inspiration from Hongkiat. This was my first time using SASS and CSS animations. This project helped me think of the web page as a dynamic canvas to do more than just place/style HTML elements. One month into development, I am excited to see where this project goes in the future.

This website has gone through multiple iterations of wireframing and prototyping, with input from many of my friends and instructors. Each effect, animation, font-size, arrangement and color choice has been carefully selected, and it will continue to develop in the future as I continue to grow as a developer and designer.

Design Points

I love minimalistic designs. Designs that make a user say "wow" are not extravagant-- they are intricate and immaculate. Find the beauty of the website in this modal's close button that rotate subtly on hover and the soft colors that pop on every view.

From the loading animation to section dividers, this website plays with the concept of the teal circle. There's no corny symbolism there-- just a design concept I found elegant :)

Shout out to my amazing mentee Elizabeth Wang for helping me sketch the project icons!

UPC Website

Jekyll

Mar 2017

I created the website for CMU's Undergraduate Poker Club as an improvement on our webmaster's previous work. This website was created to present information about the club ahead of its plans to seek corporate sponsorship. Check out the website here.

I used Jekyll to allow my other non-technical teammates to manage the website in the future. All event cards, event pages and officer information are displayed by HTML that is generated either from a central Markdown file or a collection the website's configuration file-- it is very easy for anyone to follow the template set by other Markdown post to add content to the site like a blog.

Given that most variable content can be added and deleted using non-HTML files, the base HTML/CSS code will never have to be touched other than for the purpose of redesign.

CMU Canvas Help Website

UX Design/Jekyll

Feb 2017 - Apr 2017

As a support technician for the CMU Eberly Educational Technologies Center, I was tasked with creating a CMU-specific help website for Canvas LMS.

The previous help website for Blackboard was very cluttered and unattractive. Given the large scale of the QnA topics, I focused on how to optimize the user flow by prioritizing use cases and standardizing design elements. I have created a placeholder website to direct professors who need help while the Eberly Center staff collate the appropriate support content.

The challenge with this project is meeting my department's requirements and optimizing user experience while following CMU's design standards.

Currently, I have prioritized the user stories, categorized them and drawn an iteration of wireframes. I will move onto designing mockups and developing a first draft of the website to begin user testing. I plan to develop the website using Jekyll to take advantage of its flexbility and scalability. The website should be complete by the beginning of April.
